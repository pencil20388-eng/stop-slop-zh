# 禁用结构模式

## 编号式结构

AI 极度偏爱用编号组织内容，这是最明显的 AI 写作信号。

**禁用**：
```
判断一：市场正在收缩
判断二：技术壁垒在降低
判断三：新玩家正在入场
```

**替代**：用自然过渡连接
```
市场正在收缩，这一点从 Q3 数据就能看出来。与此同时，技术壁垒也在降低……再加上新玩家不断入场，整个格局跟半年前完全不一样了。
```

**禁用**：`第一...第二...第三...`
**替代**：`一个是...另一个是...`「先...再...」或直接散文式展开

## Bullet Point 罗列

大段要点罗列是典型的 AI 输出格式。

**禁用**：
```
这个方案的优势包括：
- 成本降低 40%
- 部署时间缩短到 2 天
- 支持多语言
- 无需专业运维
```

**替代**：
```
这个方案最直接的好处是成本降低了 40%，部署时间从原来的两周压到了 2 天。多语言支持开箱即用，运维那边也不需要专人盯着。
```

## 大量加粗

AI 喜欢用加粗标注关键词，一篇文章动辄十几二十处加粗。

**规则**：全文加粗控制在 10 处以内。超过就是 AI 味。

## 不必要的小标题

AI 喜欢给每个段落加小标题，把文章切成碎片。

**规则**：绝大多数文章从头到尾顺下来，不需要小标题。只有超长内容（3000 字以上）才考虑分节。

## 「总分总」八股结构

**禁用模式**：
```
[开头总述：本文将讨论 A、B、C 三个方面]
[分述 A]
[分述 B]
[分述 C]
[结尾总结：综上所述，A、B、C 三方面表明...]
```

**替代**：开头用反差数据或具体事件切入，结尾回扣开头形成闭环，中间用自然过渡而非机械分块。

## 对比表格滥用

AI 喜欢用表格做对比，哪怕内容只有两三条。

**规则**：只有在对比维度超过 4 个且数据密集时才用表格。简单对比用散文表达。
